ğŸ“ŒCLASS MUTTABLE OR IMMUTABLE 
In C#, a class itself is not inherently mutable or immutable.
Its behavior depends on how you define it.

1. By default â€” Class is Mutable Because:
-Properties usually have setters, so values can be changed.
-Fields can be updated after object creation.

Example (Mutable Class):

public class Person
{
    public string Name { get; set; }  // can be changed
}
static void Main()
{
	var p = new Person { Name = "John" };
	p.Name = "David";   // âœ… changed after creation
}


2. Making a Class Immutable
To make it immutable:
-Use readonly fields or get-only properties.
-Set values only through the constructor.
-Donâ€™t provide any set methods.

Example (Immutable Class):
public class Person
{
    public string Name { get; }  // get-only property

    public Person(string name)
    {
        Name = name;
    }
}
static void Main()
{
	var p = new Person("John"); // p.Name = "David"; // âŒ Not allowed, no setter	
}
