ğŸ“ŒWhat is GAC?
GAC stands for Global Assembly Cache.
It is a special shared folder on a Windows machine where .NET assemblies (compiled code libraries /.dll files) are stored so they can be used by multiple applications.

Why does GAC exist?
When you build a .NET application, you often use assemblies (DLL files) â€” reusable code libraries.
Sometimes, different applications need to use the same assembly, and:
You donâ€™t want to copy that assembly into each application folder (to save space and avoid version conflicts).
You want to share a single version or multiple versions of an assembly safely.
The GAC solves this problem by providing a centralized place to store and manage these shared assemblies.

How does GAC work?
Assemblies installed in the GAC must have a strong name â€” a unique identity that includes:
The assemblyâ€™s name
Version number
Culture (language info)
A public key token (from a cryptographic key pair)
When an application runs, the .NET runtime looks in the GAC first for any assemblies with a matching strong name.
The GAC can store multiple versions of the same assembly side by side, so different applications can use different versions without conflict.

Location of GAC
On Windows, the GAC is usually located at: C:\Windows\Microsoft.NET\assembly\

How to install/remove assemblies in GAC?
Use the Global Assembly Cache Tool (gacutil.exe) from the Visual Studio Developer Command Prompt.
To install: gacutil -i YourAssembly.dll //Adding ur assembly into GAC for global use
To uninstall: gacutil -u YourAssembly //removing any assembly from GAC
