üìÅ Connected Services/
What it is: Helps you connect to services like:
Azure Storage, Azure SQL Database
External Web APIs, authentication providers, etc.
Currently empty in new projects unless you add services via Visual Studio.
Real-Life Analogy:
Think of your app like a shop. If you want to sell online, you'll connect to shipping services (FedEx) or payment gateways (PayPal). Connected Services is the section where you manage these external "partners."


üìÅ Dependencies/
What it includes:
NuGet packages (like libraries/tools)
Framework references (Microsoft.AspNetCore.App, etc.)
Project references (if you have multiple projects)
Example:
<PackageReference Include="Microsoft.EntityFrameworkCore.SqlServer" Version="7.0.0" />
Real-Life Analogy:
Imagine you‚Äôre running a coffee shop. You depend on tools like espresso machines, milk frothers, etc. These dependencies are tools or supplies without which your shop (app) can‚Äôt function.


üìÅ Properties/ ‚û§ launchSettings.json
Purpose: Defines how the app should start in different environments (Development, Production, etc.)
Includes settings like:
Launch browser?
Environment variables?
Ports to run on?
Sample:

  "MyApp": {
    "commandName": "Project",
    "launchBrowser": true,
    "environmentVariables": {
      "ASPNETCORE_ENVIRONMENT": "Development"
    },
    "applicationUrl": "https://localhost:5001"
  }
}



üìÑ appsettings.json
Main config file for:
Logging settings
Connection strings : for connecting to database
Custom settings (e.g., pagination, API keys)
EG :
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=.;Database=ShopDB;Trusted_Connection=True;"
  },
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft": "Warning"
    }
  }
}


üìÑ Program.cs
This is the entry point of the ASP.NET Core app.
Contains:
Main() method
Calls CreateHostBuilder() ‚Üí sets up web server
Adds services to DI container
Configures middleware (routing, authentication)
EG :
var builder = WebApplication.CreateBuilder(args);
builder.Services.AddControllers();
var app = builder.Build();
app.MapControllers();
app.Run();
