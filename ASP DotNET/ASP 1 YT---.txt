Normal Desgin patterm :are used for solving specific techinal issues (probklems )only, Where as an architectural pattern is used for solving larger architectural problems, So it affects entire architecture of our project

-MVC divides the whole applicaition in 3 parts , MODEL, VIEW, CONTROLLER.

MVC components are:


CONTROLLER : Client will send the req to the controller, Based on the req, the controller will talk to the Model(which is the ERM).
-Controller Takes user input, also called request paramters.
-Controller acts as the middle man between views and model.


MODEL : This will convert the classes into tables, Basically this will talk with the database, The controller will talk to this model and take the specific data which the user(clients ) wnat.
-Model Interacts with database.
-It is also responsible for executing bussness logic

VIEW : The data from the model will come to controller, Now the controller will send the data to the views(UI PAGES), Views will serialize(convert/PARSE) the data into human readable form.
-This views will send the data to the user from controller
-View is what the user sees(frintend) on the screen.
-Generates UI for the user.
