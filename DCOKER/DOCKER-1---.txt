üìåDOCKER VS GIT
üîß What Git and GitHub Do
Git is a version control system. It tracks changes to source code over time.
GitHub is a remote hosting platform for Git repositories. It helps developers:
-Share and collaborate on code
-Review changes (pull requests)
-Track issues and progress
-‚ùóGit and GitHub manage your code, not your runtime environment.

üê≥ What Docker Does
Docker is a containerization platform. It packages your application with everything it needs to run like:
-Operating system libraries
-Dependencies (like .NET, Python, Node.js, etc.)
-Configurations
-Think of Docker as a "mini virtual machine", but much more lightweight and faster.

üí° Why Docker When We Have Git/GitHub?
| Feature                                         | Git/GitHub | Docker |
| ----------------------------------------------- | ---------- | ------ |
| Version control of code                         | ‚úÖ          | ‚ùå      |
| Team collaboration & code sharing               | ‚úÖ          | ‚ùå      |
| Run code in same environment on any machine     | ‚ùå          | ‚úÖ      |
| Package code + dependencies + runtime           | ‚ùå          | ‚úÖ      |
| Ensure "it works on my machine" issues are gone | ‚ùå          | ‚úÖ      |
| Deployment-ready app environments               | ‚ùå          | ‚úÖ      |

+-------------------+       +------------------------+

|   Container 2     |       |     VM 2               |

| | App & Libs  |   |       | | Guest OS (Windows) | |
| +-------------+   |       | | App & Libs         | |
+-------------------+       +------------------------+

Docker: Lightweight, shares Host OS kernel.
VMs: Heavy, each runs its own full OS.
