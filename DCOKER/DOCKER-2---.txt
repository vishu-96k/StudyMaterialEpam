âš™ï¸ Difference between Attach mode and Detach mode in Docker
--------------------------------------------------------------------------------------------------------------
| Feature / Point                  | ATTACH MODE (Default)                                    | DETACH MODE (-d)                                   |
|----------------------------------|----------------------------------------------------------|----------------------------------------------------|
| Meaning                          | You are directly connected to the container terminal.    | Container runs in the background.                  |
| Command Example                  | docker run -it ubuntu                                    | docker run -d nginx                                |
| Terminal Behavior                | Terminal stays busy showing container output/logs.       | Terminal becomes free immediately after starting.  |
| Interaction                      | You can type and interact inside the container.          | You cannot interact directly unless you attach later.|
| When to Use                      | For testing, debugging, or learning (manual interaction).| For running long-running apps/services.            |
| How to Exit                      | Type 'exit' or press Ctrl + C to stop/leave container.   | Use 'docker stop <container_id>' to stop it.       |
| Checking Running Containers      | You only see current container activity on screen.       | Use 'docker ps' to see running containers.         |
| Example Use Case                 | Testing commands inside Ubuntu or Python.                | Running Nginx or MySQL server in background.       |
| Real-Life Comparison             | Watching a movie â€” you see everything live.              | Playing music in background â€” it runs silently.    |
