üìåSERVICE DELIVARY LEVELS :
1)saas :software as a service eg: Apllications 
-> SAAS main hame pura ka pura software he as a service milta hian
-> You use ready-made software over the internet ‚Äî no need to install or maintain anything.
The service provider manages everything ‚Äî hardware, OS, application, updates, and security.
In simple words:
You just use the app, everything else is handled by the provider.
-Examples:
Gmail ‚Äì you just log in and send emails.
Microsoft 365 / Google Docs ‚Äì you use apps online, no installation needed.

2)paas :platfrom as a service Eg: os & middleweres
-> PAAS : mian hame pura ka pura platform (computer with OS, RAM, and hardware) as a service milta hain, like OS, harddisk, RAM, and iss computer par hame hamare kudhke application build and deploy krne hain
-Isame sirf vo platfrom jo hian (harddrive ,RAM, os), that are manage by the service provider, baki jo app build nd deploy krna hian that we need to manage on our own.

‚úÖ You manage: Your applications & data.
‚úÖ Provider manages: OS, runtime, middleware, and infrastructure.

->You get a platform to build, test, and deploy your own applications.
The provider manages hardware, operating system, and middleware ‚Äî you just focus on developing the app.
-In simple words:
You build your app on someone else‚Äôs platform without worrying about servers or OS setup.

Examples:
Microsoft Azure App Service
Google App Engine
AWS Elastic Beanstalk


3)Iaas :Infrastruture as a service EG : HW and Building infrastructure
-IAAS : main ham sirf infrastructure lete hian, mtlb empty computer (sirf body), baki uss computer main konsa OS chalega, Kitana ram rahega, konse applications install krne hain, konsa app new build krna hian, kaha deploy krna hian, ye sab ham kudh handle krenge.
-IAAS : main vender sirf infra manage krega, mtlb khali computer ke hardware 

‚úÖ You manage: OS, runtime, app, data.
‚úÖ Provider manages: Physical hardware, virtualization, and networking.

You rent virtual hardware and network resources like servers, storage, and networking from a provider.
You manage the OS, applications, and data yourself.
-You rent the infrastructure and build everything else on top of it.
Examples:
Amazon EC2 (AWS)
Microsoft Azure Virtual Machines


| Level    | Who Manages What                                | What You Get               | Example           |
| -------- | ----------------------------------------------- | -------------------------- | ----------------- |
| **SaaS** | Provider manages everything                     | Ready-to-use software      | Gmail, Zoom       |
| **PaaS** | Provider manages platform; you manage app       | Platform to build apps     | Google App Engine |
| **IaaS** | Provider manages hardware; you manage OS & apps | Virtual machines & storage | AWS EC2           |
