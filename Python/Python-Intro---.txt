2)PROPERTIES :

1. Mutability
List: You can add, remove, or update elements.
Tuple: Fixed after creation. You can‚Äôt modify, but can contain mutable objects inside.
Dict: Keys must be unique and hashable; values can be changed.
Set: You can add/remove elements, but can‚Äôt store mutable elements like lists.


2. Performance (Speed)
| Operation Type          | Fastest Type      | Reason                                                    |
| ----------------------- | ----------------- | --------------------------------------------------------- |
| Iteration               | `tuple`           | Fixed size, less memory overhead                          |
| Lookup by index         | `tuple` or `list` | Both O(1)                                                 |
| Lookup by key           | `dict`            | Hash table O(1) average case                              |
| Membership check (`in`) | `set` / `dict`    | Hash lookup is much faster than linear scan in list/tuple |

Why tuple is faster than list:
Tuple is immutable, so Python can optimize memory and iteration.
List allows modifications, so it keeps extra memory (overhead) to allow dynamic resizing.


3. Internal Working
| Type    | Internal Implementation           | How it works                                                          |
| ------- | --------------------------------- | --------------------------------------------------------------------- |
| `list`  | Dynamic array (array of pointers) | Elements stored in contiguous memory; resizing happens automatically. |
| `tuple` | Fixed array of pointers           | Immutable, so stored more compactly.                                  |
| `dict`  | Hash table (key ‚Üí value mapping)  | Keys hashed to index location; O(1) average access.                   |
| `set`   | Hash table (only keys, no values) | Like dict without values; ensures uniqueness of elements.             |


4. When to Use Which
| Use Case                                | Best Data Type |
| --------------------------------------- | -------------- |
| Changeable sequence of items            | `list`         |
| Fixed sequence (no modification)        | `tuple`        |
| Key-value pairs, fast lookups           | `dict`         |
| Unique elements, fast membership checks | `set`          |
