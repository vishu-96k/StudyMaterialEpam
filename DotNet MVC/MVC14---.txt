âœ…[#1]What is Dependency Injection (DI)?
Dependency Injection is a design pattern that deals with how components acquire their dependencies. Rather than creating objects directly, DI provides them from an external source. So we will be making lossly coupled objects, by sending the objects through constructor injections, property injectsions, or through function injection.
 Goal:
To achieve loose coupling between classes and their dependencies, improve testability, and follow the SOLID principles, especially the D - Dependency Inversion Principle.

Step 3: Configure Unity in Application_Start
App_Start/UnityConfig.cs

using Microsoft.Practices.Unity;
using System.Web.Mvc;
using Unity.Mvc5;

public static class UnityConfig
{
    public static void RegisterComponents()
    {
        var container = new UnityContainer();

        // Register types
        container.RegisterType<IStudentService, StudentService>();

        DependencyResolver.SetResolver(new UnityDependencyResolver(container));
    }
}

Step 4: Use DI in Controller
Controllers/StudentController.cs
ðŸ‘‡ Code:
public class StudentController : Controller
{
    private readonly IStudentService _studentService;

    public StudentController(IStudentService studentService)
    {
        _studentService = studentService;
    }

    public ActionResult Index()
    {
        var students = _studentService.GetAllStudents();
        return View(students);
    }
}
//ASP.NET MVC will automatically resolve the IStudentService and inject StudentService.

Types of DI:
| Type                  | Description                                   | Example                                     |
| --------------------- | --------------------------------------------- | ------------------------------------------- |
| Constructor Injection | Dependencies are injected via constructor.    | `public MyClass(IMyService svc)`            |
| Property Injection    | Dependencies are set via public properties.   | `public IMyService MyService { get; set; }` |
| Method Injection      | Dependencies are passed to a method directly. | `DoSomething(IMyService service)`           |
