ðŸ“ŒWhat is MVC Architecture?
MVC stands for Model-View-Controller â€” a design pattern that separates an application into three main components:

Model
Represents the data and business logic.
Deals with data access, validation, and rules.
Example: Classes that represent database entities, or data access layers.

View
Represents the UI (user interface).
Displays data to the user and sends user commands to the controller.
Example: Razor views (.cshtml files) that render HTML.

Controller
Handles user requests.
Interacts with Model to process data and selects a View to display the result.
Example: Controller classes with action methods that respond to URL routes.

Why MVC?
Separation of concerns: Each component handles a specific responsibility, making code easier to maintain and test.
Testability: You can test each part independently.
Flexibility: Different views can represent the same data (e.g., web, mobile).


How MVC works in ASP.NET Core MVC?
User makes a request (e.g., /Home/Index).
The Routing system routes this to the appropriate Controller (e.g., HomeController).
The Controller interacts with the Model to fetch/process data.
The Controller returns a View with a Model.
The View generates HTML and sends it to the user's browser.

Step 3: Create the Controller
Create a folder named Controllers (if not existing) and add ProductController.cs:
ðŸ‘‡ Code :
using Microsoft.AspNetCore.Mvc;
using MvcDemoApp.Models;
using System.Collections.Generic;

namespace MvcDemoApp.Controllers
{
    public class ProductController : Controller
    {
        // Normally, you'd get data from a database or service.
        // For simplicity, we'll use a hard-coded list here.
        private static List<Product> products = new List<Product>
        {
            new Product{ Id = 1, Name = "Laptop", Price = 1200 },
            new Product{ Id = 2, Name = "Smartphone", Price = 800 },
            new Product{ Id = 3, Name = "Tablet", Price = 500 }
        };

        public IActionResult Index()
        {
            return View(products);  // Pass model (list) to the view
        }

        public IActionResult Details(int id)
        {
            var product = products.Find(p => p.Id == id);
            if (product == null)
                return NotFound();

            return View(product);
        }
    }
}
