âœ… What is a Controller in ASP.NET MVC?
ðŸ“Œ Definition:
ðŸ”· A Controller is a class in ASP.NET MVC that handles user requests, processes them (often by interacting with the model), and returns a response (typically a View or data UI).

âœ… What Does a Controller Do?

ðŸ”· Accepts HTTP Requests from the browser (like GET, POST).

ðŸ“Œ Request Handling Flow:

ðŸ“Œ Internally:
ðŸ”· ASP.NET MVC uses Routing to map URL paths to controller actions.

âœ… Example of a Controller in ASP.NET MVC:
ðŸ“Œ C# Code Example:

{
    // Action to show a list of products
    public ActionResult Index()
    {
        var products = ProductRepository.GetAll(); // Get data from model
        return View(products); // Pass data to view
    }

    // Action to show details of a specific product
    public ActionResult Details(int id)
    {
        var product = ProductRepository.GetById(id);
        if (product == null)
        {
            return HttpNotFound(); // Return 404 if not found
        }
        return View(product); // Pass product to view
    }

    // Action to handle form submission
    [HttpPost]
    public ActionResult Create(Product model)
    {
        if (ModelState.IsValid)
        {
            ProductRepository.Add(model);
            return RedirectToAction("Index");
        }
        return View(model); // Return to the form if validation fails
    }
}

âœ… Understanding ASP.NET MVC Routing
ðŸ“Œ Definition:
ASP.NET MVC Routing is a mechanism that maps incoming browser requests to specific controller actions. This allows for clean, user-friendly URLs that are not tied to physical files on the server 

âœ… How Routing Works in ASP.NET MVC
ðŸ“Œ Core Components:
ðŸ”· Route: Defines a URL pattern and the handler (controller and action) for that pattern.
ðŸ”· Route Table: A collection of routes that the application uses to match incoming requests.
ðŸ”· Routing Engine: Processes incoming requests and matches them to the appropriate route in the route table.


ðŸ“Œ Default Route:
By default, ASP.NET MVC includes a route defined as:
EG :
routes.MapRoute(
    name: "Default",
    url: "{controller}/{action}/{id}",
    defaults: new { controller = "Home", action = "Index", id = UrlParameter.Optional }
);

ðŸ”· Explanation:
{controller}: Specifies the controller name.
{action}: Specifies the action method within the controller.
{id}: An optional parameter that can be used to pass additional data
For example, a URL like /Product/Details/5 would be mapped to the Details action of the Product controller with an id of 5.


âœ… Types of Routing in ASP.NET MVC
ðŸ“Œ 1. Convention-Based Routing
ðŸ”· Routes are defined in a centralized location, typically in the RouteConfig.cs file.
ðŸ”· Follows a consistent pattern across the application

ðŸ“Œ 2. Attribute Routing
ðŸ”· Introduced in ASP.NET MVC 5, allowing developers to define routes directly on controller actions using attributes.
ðŸ”· Provides more control and flexibility over URL patterns.
EG :
[Route("products/{id}")]
public ActionResult Details(int id)
{
    // Action logic here
}
This route would match URLs like /products/5 and direct them to the Details action.

âœ… Customizing Routes
ðŸ“Œ Defining Custom Routes:
You can define custom routes to match specific URL patterns:
routes.MapRoute(
    name: "CustomRoute",
    url: "store/{category}/{id}",
    defaults: new { controller = "Store", action = "Browse", id = UrlParameter.Optional }
);
This route would match URLs like /store/electronics/5 and direct them to the Browse action of the Store controller with category as "electronics" and id as 5.

âœ… Benefits of Using Routing
ðŸ”· Clean URLs: Provides user-friendly and SEO-friendly URLs.
ðŸ”· Decoupling: Separates URL structure from the physical file structure.
ðŸ”· Flexibility: Allows for complex URL patterns and parameter passing.
ðŸ”· Maintainability: Centralized route definitions make it easier to manage and update URL patterns.
