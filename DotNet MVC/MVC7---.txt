-View Discovery in asp .net MVC core is a process by which the freamwork determines which view file to use when rendering a response to HTTP request
-This process (of view discovery) involves locating appropriate view file based on the action method and controller.

ğŸ“ŒUnderstanding View() Method 
-This view() method is in the Controller base class, you will find four overload versions of the View() method available in Controller base class.

View() Method Overloads
-In ASP.NET Core MVC, the View() method of the Controller class has 4 main overloaded versions:

| Method Signature                      | When to Use                                                  |
| ------------------------------------- | ------------------------------------------------------------ |
| `View()`                              | Return a view with **same name as action**; no data passed.  |
| `View(object model)`                  | Return a view with **same name as action**; pass data to it. |
| `View(string viewName)`               | Return a view with a **custom name**; no data passed.        |
| `View(string viewName, object model)` | Return a view with a **custom name** and pass data to it.    |


ğŸ¯ Real-Life Analogy
Imagine you're building custom invitations for different people.
View() â†’ You use a default template because the person's name matches the invitation file (e.g., "John" uses John.cshtml).
View(object model) â†’ You use a default template but add personal details like a message or address.
View("Birthday") â†’ You manually select a template called "Birthday" instead of using "John" even though the request came from John.
View("Birthday", personData) â†’ You use a "Birthday" template with Johnâ€™s personalized data (name, age, etc.).
